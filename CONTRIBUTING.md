# Contributing to Boilermaker

This document is for people wanting to contribute to the Boilermaker project. We welcome contributions of any kind, including bug fixes, new features, documentation improvements, and more. If you're interested in contributing, please read the guidelines below to get started.

## Getting Started

First things, first, please have the following:

- A Github account
- A willingness to learn and contribute to an open-source project for the betterment of the project and the community.
- A relative working knowledge of Git and Github, or a willingness to learn. This includes forking repositories, creating branches, making commits, and submitting pull requests.

### Getting an Invite

While we may review and accept contributions from non-contributors, we ask that you please request to be added as a contributor to the project. This allows us to better manage contributions and ensure that everyone has the necessary permissions to contribute effectively. To request access, please open an issue in the [Boilermaker repository](https://github.com/yeajustmars/boilermaker/issues) with the following information:

- Title: "Request to join Boilermaker"
- Your Github username
- Something brief about you and your interest in contributing to the project.
- Any relevant experience or skills you have that would be beneficial to the project (e.g., programming languages, open-source contributions, etc.)
- Whether you are interested in being a core maintainer, template creator, writing documentation or if you're open to contributing in any way.

## Pull Requests

Please do not submit a Pull Request until you have been approved as a contributor and assigned to an Issue. We don't do this to be draconian but instead to ensure the continued quality of the codebase and to make sure that contributions are aligned with the project's goals and standards. Unsolicited PRs may be politely closed. This is to ensure a level of code quality and consistency across the project, as well as to continue to guarantee that official templates are ready to run, out of the box!

### Creating Pull Requests

1. Fork the repository and create a new branch for your changes.
    1. The branch should be based on the latest version of the `main` branch.
    2. The branch should be explicit about what it's doing. For example, "fix: #123" or "feat: improve logging in the CLI"
2. Make your changes in the new branch.
3. Commit your changes with a clear and descriptive commit message.
4. Push your chanages to your forked repository.
5. Open a Pull Request from your branch to the `main` branch of the original repository.
6. In the Pull Request description, please include:
    - A clear and concise description of the changes you made.
    - The issue number(s) that your changes address (if applicable).
    - Any relevant screenshots or documentation updates (if applicable).
7. Submit the Pull Request and wait for a maintainer to review it.
